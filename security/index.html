Security Assessment of a Ride Hailing Service

Written by Devki Kalra

Introduction

The purpose of this lab is to identify and examine potential flaws and vulnerabilities with the web application hosted at https://jordan-marsh.herokuapp.com/. This application was created by Ming Chow for the security lab of the COMP120 course. The app's function is to allow users to enter their username, longitude, and latitude. A number of security flaws found in the source code make this application unstable and unusable for public use.

Methodoly

I used curl on my local device. I tested the service by changing the parameters and checking the output on https://jordan-marsh.herokuapp.com/. I likewise used Burp to find additional vulnerabilities and/or gain a deeper understanding of the vulnerabilities I had already identified.

Abstract of Findings

The biggest issues found in this security assessment concerned XSS and SQL injections. Each of the issues detected were high in severity and thus expose the massive vulnerabilities in this application. User input wasn't effectively sanitized allowing scripts to be stored and executed from the server, and poor restrictions around database access meant the contents of the database could be easily manipulated.

Issues Found

  1. XSS Injection
      2. Issue: Injection into username parameter
      Location: https://jordan-marsh.herokuapp.com/
      Severity: HIGH
      Description: The username parameter can take in any input. There is no such restriction or validator for the username (like there is for the lat and lon values, as these must be floats). As a result, JavaScript code could be passed in as the username parameter. This type of injection is referred to as an XSS injection.
      Resolution: This issue can be resolved by sanitizing the input. This could be done through setting a limit on the number of characters that can be entered in the username parameter.
  2. SQL Injection
      3. Issue: Injection into database
      Location: https://jordan-marsh.herokuapp.com/
      Severity: HIGH
      Description: This issue was detected on Burp and arises when user-controllable data is incorporated into database SQL queries. With this web application, users can easily manipulate the database with a curl request. Nonetheless, I decided to include it in my report because I understand the vulnerability.
      Resolution: According to the issue remediation outlined in Burp, the most effective way to prevent SQL injection is to use 'parametized queries' for all database access. Meaning, the application should specify both the structure of the query and contents of the variable. In this way, it is impossible for malformed data to be injected into the database.
